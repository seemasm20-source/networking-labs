Issue - PC's Could Not Communicate Between LANs





📋 Problem Summary:-



All PCs across both LANs had no default gateway configured. This blocked all cross-LAN communication


between HR (LAN 1) and Finance (LAN 2) while same-LAN communication remained unaffected.








🔍 Cause:-



The default gateway field was left empty on all PCs during initial IP configuration. Without a gateway the PC has no exit route to send


traffic outside its own LAN, it cannot reach the router or any other network.


PC → Desktop → IP Configuration

IP Address   : 10.0.0.1       ✅ set


Subnet Mask  : 255.255.255.0  ✅ set


Default GW   : (empty)        ❌ missing





What is a Default Gateway?

The default gateway is the router's IP address on that PC's side of the network. It is the PC's exit door to reach other networks.

Without it the PC is stuck inside its own LAN.

| Test                                    | Result      | Reason                                      |
| --------------------------------------- | ----------- | ------------------------------------------- |
| Same-LAN ping (10.0.0.1 → 10.0.0.2)     |  Reply ✅    | Switch handles same-LAN  gateway not needed |
| Cross-LAN ping (10.0.0.1 → 192.168.1.1) | Timed out ❌ | Must exit through router gateway missing    |






























































Key learning:


Gateway is only needed when traffic leaves the local network. Same-LAN traffic goes through switch directly



gateway is never consulted. Cross-LAN traffic must exit through the router  gateway is mandatory.

🔧 Fix :-


Added the correct default gateway on all PCs:





Click each PC → Desktop → IP Configuration

LAN 1 PCs (PC1, PC2, PC3):
  Default Gateway → 10.0.0.4     (Router Gi0/0 IP)

LAN 2 PCs (PC4, PC5, PC6):
  Default Gateway → 192.168.1.4  (Router Gi0/1 IP)





The gateway must always match the router's
interface IP on that PC's side of the network.

🎯 Real-World Help Desk Ticket

User complaint:


I can reach my colleagues on the same floor but cannot reach anyone in the Finance department 

or access any shared servers.


Root cause:

Default gateway was missing on the user's PC. Same-LAN communication worked because it goes


through the switch directly — no gateway needed. Cross-LAN communication failed because the PC

had no exit route to reach the router.


Resolution:


Added correct default gateway on all PCs. LAN 1 PCs → 10.0.0.4 , LAN 2 PCs → 192.168.1.4


Verified with cross-LAN ping test.
